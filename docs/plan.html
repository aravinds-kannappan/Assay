01

The thesis: most eval deltas are inside their own noise floor

Named for the signal-processing term. The noise floor is the level below which a measurement cannot be trusted. Ship the number without it and you report noise as signal.

Every major benchmark has documented, measurable flaws, and almost no eval tool reports uncertainty. Of twelve widely used eval tools surveyed, exactly one implements clustered standard errors and zero implement paired significance tests or power analysis. Harnesses print bare point estimates; leaderboards rank on them; papers claim 2-point wins on 200-item sets that cannot resolve anything smaller than six.

Noisefloor does not ship runner number thirteen. It ingests the per-sample logs Inspect AI, lm-evaluation-harness, and HELM already emit, normalizes them into one parquet schema (item id, rendered-prompt SHA256, score, cluster key, base-model family, seed, harness version, dataset revision), and makes every downstream number replayable and honestly bounded. The project is built the way it tells others to build: its own regression suite is a reproduction study, its own competitive claims ship as a dated auditable table, and every literature number it quotes is re-derived in-repo with confidence intervals, where a failed replication is published as a finding, not buried.

deterministic statistically estimated trained-model LLM-judged

Every number in every Noisefloor report carries exactly one of these four provenance tags, enforced by a schema test in CI. Nothing claims more than its method supports.

03

Architecture: thirteen modules, honestly labeled

A layer, not a runner. Deterministic core carries the product; the statistical and trained tiers earn their place or get cut.

nf-ingestdeterministic
Adapters for Inspect .eval, lm-eval samples JSON, and HELM display_predictions into one parquet schema. Prompt-SHA256 and revision pinning make every number replayable. Version-pinned conformance fixtures across the last 3 releases per harness; parse failures are loud, never silent.
nf-checkstatistical
The stats engine: CLT and CR2 cluster-robust SEs with wild-cluster-bootstrap fallback under 30 clusters, cluster-aware paired tests, unbiased Chen et al. pass@k via two-level bootstrap, MDE calculator, Holm and Benjamini-Hochberg corrections.
nf-powerdeterministic
Pre-flight gate: given n and design, prints the smallest gap the eval can resolve and the cheapest paired design that could resolve your claimed effect, with a projected item budget. The conditioning assumption is always printed, never hidden.
nf-gatedet + stat
GitHub Action. Annotates PRs with a significance verdict: claimed delta, cluster-aware paired result, MDE at current n, and the item budget that would make the claim testable. Report-only by default; failing the build is opt-in.
nf-recountstatistical
Ranking-flip engine. Given original vs corrected item sets, recomputes every affected pair's delta under cluster-aware paired bootstrap and outputs a flip probability per adjacent leaderboard pair. The pair scan is Holm-corrected, because cherry-picking one flip from thousands is the forking-paths error the tool polices.
nf-fasttrained-model
2PL IRT (~62k params) fit by Adam on Bernoulli NLL over Open LLM Leaderboard outcomes. Yields item difficulty/discrimination, model ability θ with credible intervals, and Fisher-information subsets sized to a target MDE. Every subset ships its θ applicability range and an extrapolation warning.
nf-audit3 tiers
Tiered benchmark linter: (1) deterministic lint for degenerate items, (2) statistical flags from cross-model disagreement + IRT misfit, validated by difficulty-matched hypergeometric enrichment against a cleanlab baseline, (3) experimental trained classifier behind a flag with a kill criterion. LLM triage is quarantined and always tagged.
nf-reconcilereplay + budget
Cross-harness reconciler. Deterministic core replays one frozen generation set through competing extraction and scoring paths and attributes each point of delta to a named implementation choice. Budgeted tier runs fresh inference only on models ≤ 8B for logprob-scored paths.
nf-probestatistical
Construct-validity shortcut probes: question-blind and answer-only baselines runnable on any MC benchmark, reporting how much of the metric surface features alone recover, with CIs. The HellaSwag (~68%) and TruthfulQA (79.6%) shortcut figures are re-derived fresh, not quoted.
nf-judgestatistical
Judge validation: Cohen κ and Krippendorff α vs human labels (arena votes labeled crowd-preference, not gold), position-bias flip rate under order swap, verbosity coefficient via logistic regression. An agreement bar with published sensitivity analysis gates any LLM judge before it may feed nf-gate.
nf-watchstatistical
Anytime-valid early stopping: betting confidence sequences (Waudby-Smith & Ramdas) stop an expensive sweep the moment the delta CI excludes zero, with exact α under continuous peeking. Items pre-shuffled for exchangeability; stopped point estimates flagged as winner's-curse biased.
nf-reportdeterministic
Single-file static HTML report: error bars, paired verdicts, power curves, flag lists, harness-diff attribution tables. A schema test in CI fails the build if any rendered number lacks its provenance tag, so honest labeling is machine-enforced.

05

The statistics is load-bearing, not decorative

Each method is here because ignoring it produces a wrong conclusion in a specific place in the pipeline. This is the section a skeptical statistician reads first.

CR2 cluster-robust SEs+ wild cluster bootstrap under 30 clusters
where → default CI width in every reportNaive SEs understate DROP uncertainty by 3.05× (Miller Table 4), which flips gate verdicts. CR0 is anti-conservative with few clusters, exactly the SWE-bench (~12 repos) and MMLU (57 subjects) regime. A metrology tool that understates its own SEs commits the sin it audits.
Cluster-aware paired inferenceCRSE on paired diffs; cluster bootstrap of McNemar
where → the canonical pass/fail in nf-gate and nf-recountPairing makes 2-point deltas testable at ~2,000 items instead of ~9,800 (conditional on ~10% disagreement, always stated). Independent-pairs McNemar on clustered items inflates Type I. Making the paired test cluster-aware removes the internal contradiction a judge flagged as the concept's core flaw.
Power analysis / MDEMiller Eq 9, factor 7.85 at α=0.05, power=0.8
where → printed by nf-power before every runThe product's core promise: telling users what their eval cannot see. A 200-item eval that cannot resolve less than a 6-point gap should say so before tokens are spent.
Two-level hierarchical bootstrapunbiased Chen et al. pass@k estimator
where → CIs on the HumanEval+/MBPP+ weak-test auditsThe naive pass@k estimator is biased and single-level resampling ignores completion-within-problem structure. Ranking-flip claims about EvalPlus deflation are only credible with correct intervals.
Holm + Benjamini-Hochbergcorrections on every multi-slice scan
where → nf-gate slice reports, nf-recount pair scansAt 30 uncorrected slice tests, 1.5 spurious wins are expected at α=0.05. Selecting a showcase flip from thousands of leaderboard pairs without multiplicity control is the forking-paths error a reviewer catches in the first meeting.
Simultaneous inference for tie groupsconnected components, labeled non-transitive
where → the Indistinguishable Leaderboard demoPairwise indistinguishability is not transitive, so "collapse ties into classes" has no unique answer. Reporting connected components of the not-significantly-different graph with an explicit intransitivity label makes the demo defensible instead of ill-defined.
2PL IRT + Fisher-information selectionpublished θ applicability ranges
where → nf-fast subsets and ability reportingAccuracy stops discriminating at the top; IRT ability keeps discriminating, and information-optimal subsets hit a target MDE at minimum item cost. The applicability range addresses the documented frontier-θ drift failure of fixed subsets.
Hypergeometric enrichmentdifficulty-matched null + cleanlab baseline
where → validation of every nf-audit flag tierKnown label errors concentrate in hard items, so an exchangeable null passes even if flags only track difficulty. The difficulty-matched null closes that confound; the cleanlab baseline answers the first question any reviewer of label-error work asks.
Anytime-valid confidence sequencesWaudby-Smith & Ramdas + winner's-curse flag
where → nf-watch early stoppingRepeated peeking at fixed-n tests inflates Type I (re-derived in-repo by simulation, not quoted). Confidence sequences fix coverage but require exchangeability (hence shuffling) and leave the stopped estimate biased (hence the flag). Both caveats are printed, not footnoted.
Chance-corrected agreementCohen κ, Krippendorff α, position-flip, verbosity β
where → nf-judge admission barRaw agreement is inflated by class imbalance and style confounds. A judge that flips its verdict when answer order swaps is measuring position, not quality, and must be caught before it gates anything.
06

Two genuinely trained models, both single-GPU, both falsifiable

Real gradient descent on real data, with leakage-aware validation and a pre-registered kill criterion. A documented null result is a deliverable, not a failure.

2PL Item Response Model

trained-model
Params
~62k (θ per model; difficulty b, discrimination a per item), custom PyTorch
Data
~1,200 models × 28,659 items (~34M real binary outcomes), streamed with column projection, deduped to base-model families before any split
Objective
Bernoulli NLL, N(0,1) prior on θ, log-normal prior on a, fit by Adam. Minutes to hours on one Colab GPU, CPU-feasible
Validation
Hold out entire base-model families (never random cells, never random models, because the leaderboard is full of fine-tune siblings). Expected family-holdout per-response AUC 0.78–0.85; 100-item reconstruction MAE 1–2 pts, anchored to metabench (<0.9%) and tinyBenchmarks (~2%)
honesty note · AUC is stated lower than the random-model figure precisely because family holdout is harder. Item params published with the caveat that they are partly artifacts of the v1 harness version, cross-referenced to nf-reconcile.

DeBERTa-v3 label-error classifier

experimental
Model
DeBERTa-v3-base (86M), two per-construct heads: wrong-gold-label; underspecification. Never one pooled model
Data
Label head: ~508 positives (~370 MMLU-Redux, ~120 GSM8K-Platinum, 18 HumanEval+) + difficulty-matched negatives. Underspec head: ~1,160 SWE-bench annotated positives. Treated as positive-unlabeled learning
Objective
Class-weighted BCE per head; single GPU/Colab, <2 hrs each
Validation
Leave-one-benchmark-out within construct as the headline. GSM8K-Platinum disagreement circularity disclosed. Pre-registered ranges: text-only AUROC 0.60–0.70, precision@50 of 30–50%
kill criterion · ships only if precision@50 ≥ 30% on held-out gold AND beats both the cleanlab baseline and the disagreement+IRT-misfit statistical tier. Otherwise it is cut and the null result is published. Either outcome is a deliverable.

Risks, named with mitigations

The judges pushed hardest here. Each risk below is a flaw an earlier draft actually had, now with a concrete defense.

Scope overrun
10 weeks solo is tight. Fix: a declared cut line (trained flagger → judge module → leaderboard demo), v0.1 at week 2, each phase independently valuable. The cold-start predictor, contamination testbed, and LiveBench study are already moved to v2.
Adapter rot
Log formats churn; a silent bad parse yields confidently wrong CIs. Fix: version-pinned conformance fixtures, CI matrix over 3 releases, unknown versions fail loudly.
Trained classifier lands near chance
The 0.55-AUROC scenario. Fix: pre-registered kill criterion, per-construct heads, deterministic + statistical tiers carry the product regardless, published null is the honest deliverable.
Reconciler compute
Logprob-scored paths need fresh inference; 65B can't run on a hobby budget. Fix: deterministic replay is the core and needs no GPU; fresh inference capped at ≤ 8B, $150 total; 65B numbers labeled anchors only.
Data-engineering slog
Streaming leaderboard repos hits HF rate limits and schema drift. Fix: column projection (25–30 GB not 1 TB), resumable manifest, 1,200-repo stratified sample, 2 buffer days.
Model-lineage leakage
Fine-tune siblings across train/test inflate IRT validation. Fix: holdout by base-model family with the map published; expected AUC pre-registered lower so the honest number doesn't read as a miss.
Imported numbers fail to replicate
HellaSwag 36% traces to one blog; TruthfulQA needs fresh derivation. Fix: every literature number framed as a hypothesis; failed replication published as a finding, which strengthens a skepticism-branded project.
Adoption psychology
A bot that fails most small internal evals gets uninstalled in a week. Fix: report-only default, opt-in gating, and the annotation prescribes the exact item budget so the message is actionable.

Why this lands with Magic's Evals team

The job description, bullet by bullet, against what Noisefloor actually ships.

Build and maintain an evals platform
Platform engineering end to end: log adapters with pinned conformance fixtures, a unified schema, a CLI, a GitHub Action, a static dashboard, PyPI packaging, docs, deliberately layered on Inspect / lm-eval / HELM. The one-pip-install-on-existing-files wedge is a platform decision.
Design and validate eval tasks
nf-audit does task validation with measured precision/recall against real relabeled gold, under a difficulty-matched null and against a cleanlab baseline. Correctness is never asserted, always scored.
Infrastructure for large-scale evals and data ablations
Streams a 34M-outcome slice of the largest public per-sample corpus with a resumable manifest; the BH-corrected slice report is the ablation readout; nf-watch early-stops sweeps with exact α, cutting ablation compute without peeking inflation.
Measuring dataset quality
Quantified per-benchmark error rates with CIs, a tiered flagger whose trained tier lives or dies by a kill criterion, and first-ever ranked candidate lists for never-audited benchmarks with human-verified upstream issues as receipts.
Eval correctness, reproducibility, reliability
Prompt-SHA256, seed, harness-version and dataset-revision capture on every record; golden tests that must reproduce published statistical results before any release; every writeup rerunnable from one make target with a published spend log.
Auditing public benchmarks and OSS eval implementations
The flagship deliverables are reproduced, quantified audits: the GSM8K gap attributed regex by regex, the MMLU spread split into replayable and inference-requiring parts, EvalPlus deflation with correct CIs, plus nf-reconcile as a reusable tool no harness ships today.
Defining metrics that reflect model quality
IRT ability with credible intervals where accuracy saturates, ranking-flip probability replacing bare point deltas, judge admission gated on chance-corrected agreement plus position-flip rate.
Skepticism toward unreproducible results
nf-gate flags (and optionally fails) PRs whose gains sit below the eval's noise floor; Holm/BH kill cherry-picked slice wins; the project applies the same skepticism to itself by re-deriving every number and publishing its competitive survey as a dated auditable table.
Knowing whether a measurement captures what it claims
nf-probe operationalizes construct validity as a runnable command; the difficulty-matched null does the same for the flagger's own validity claim; and every output carries a provenance tag so no number claims more than its method supports.
